## 1) Discovery / Search Flow (Core UX)

### 1.1 Make Discovery “One Click” / More Intuitive
**Current:** discovery is a button that starts background processing, but the flow isn’t obvious to new users.

**Improvements:**
- **Add a “Run Discovery” wizard:** show current config (keywords count, region, language, min subs, max subs, quota status) and a “Start” button.
- **Add confirm dialog** (to avoid accidental long runs).
- **Show “next steps” after completion** (e.g., “Click here to view leads” / “Download CSV”).

**Where to change:**
- `templates/index.html` (UI layout)
- `static/script.js` (trigger + status updates)

---

### 1.2 Discovery Progress Should Be More Informative
**Current:** shows percent + small stats, but doesn’t show:
- current keyword being processed
- estimated time remaining
- API quota consumption in real time

**Improvements:**
- Add “Currently searching keyword…” section.
- Show **API units spent per step** (e.g., “Search: 100 units, Channels: 20 units”).
- Add a **cancel button that is smarter** (shows “canceling…” until thread stops).

**Where to change:**
- `services/discovery_service.py` → add fields like `current_keyword`, `quota_spent_this_run`
- `routes/dashboard.py` → include those fields in `/api/discovery/status`
- `static/script.js` → render the fields in the discovery section.

---

### 1.3 Better Feedback When Discovery Fails
**Current:** error modal appears when an error is detected, but it can be a generic window.

**Improvements:**
- Show a **“What to do next” checklist** based on error type (quota, auth, network).
- Add an option to **download the error log** or “Copy error details”.
- Provide a **quick fix button** (e.g., open credentials modal if auth fails).

**Where to change:**
- `services/discovery_service.py` → expand `error_details` with “recommended action” steps and a `copyable` raw message
- `templates/index.html` → add “copy error” button to modal
- `static/script.js` → implement copy-to-clipboard & conditional guidance.

---

## 2) Lead Table Improvements (Core Data Display)

### 2.1 Expand Columns + Add Sorting + Filtering
**Current:** table displays email, channel name, and contact status only.

**Improvements:**
- Add columns: **Subscribers, Avg Views, Score, Niche, Last Updated, Country**, **Last Email Sent Date**.
- Add:
  - **Column sorting** (click header to sort)
  - **Multi-filters** (e.g., score > 70, niche = “tech”, country = “US”)
  - Filter tags (like “Active”, “Inactive”, “Has Email”, “Missing Email”, “High Score”)

**Where to change:**
- `services/lead_services.py` → currently only loads leads; expand to include more fields from CSV/DB.
- `data/leads.csv` / `exporters/csv_exporter.py` → include additional columns.
- `templates/index.html` → update table with new columns.
- `static/script.js` → add filtering/sorting logic (in `loadLeads` + `renderTable`).

---

### 2.2 Row Expansion / Detail Pane
**Current:** rows are flat, no way to see detailed channel info.

**Improvements:**
- Allow clicking a row to **expand details**, showing:
  - full about/description snippet
  - last video title + date
  - full email list (if multiple extracted)
  - full scoring breakdown
  - channel link, view count, subscriber count, last upload date
- Option to view the channel on YouTube with a button.

**Where to change:**
- Extend `lead` data to include these fields.
- Add expand/collapse row in `templates/index.html` (accordions or modal).
- Add in `static/script.js` render logic for expanded view.

---

### 2.3 Data Export / Download from UI
**Current:** exports are generated in backend but not downloadable directly from UI.

**Improvements:**
- Add a **“Download CSV”** button (and optionally Excel / JSON) that downloads the latest `data/leads.csv`.
- Add optional **“Export filters”** (export only `pending`, `high-score`, etc.).

**Where to change:**
- Add new backend endpoint `/api/leads/export` returning file download.
- Update UI with a button and handler.

## 3) Keywords / Templates Management UX

### 3.1 Improve Keywords Editor UX
**Current:** keyword editing is a textarea where you edit lines.

**Improvements:**
- Show them as tags/chips (with delete buttons).
- Add “Add keyword” input field for quick adding.
- Validate duplicates and empty entries automatically.

## 4) Email Sending UX Enhancements

### 4.1 Campaign UI (progress / pause / resume)
**Current:** send logic exists but UI for “pause” / “stop” is not wired (or incomplete).

**Improvements:**
- Implement real pause/resume with a backend state tracker, not just a client-side `sendState` object (so it persists if page refreshes).
- Add **per-email status updates** (sent, failed, retrying).
- Add **estimated time remaining**, current speed (emails/minute).
- Provide a **“Retry failed”** button for errors.

**Where to change:**
- Add backend state store (probably in DB or in-memory global) to track sending status.
- Enhance `modules/email_sender.py` or create an **EmailCampaignService** that can be polled.
- Update UI in `static/script.js` to poll status and update progress panel.

---

### 4.2 Improve “sent” tracking + visibility
**Current:** stored in `data/sent_log.csv`, only email per line.

**Improvements:**
- Show **full email history** in the UI (what template was used, when sent).
- Add pagination + filtering (e.g., show “last 30 days”, show “failed”).
- Show **open/click tracking** (once implemented).

**Where to change:**
- Expand `sent_log.csv` to include timestamp, template, status.
- Update services (email_services.py, `modules/email_sender.py`) to save those details.
- Add new dashboard table for “Sent History”.
